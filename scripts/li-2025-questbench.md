[S] Score one benchmark: a model that gets ninety-one percent on grade-school math and then seven point five three percent on a block-stacking puzzle, from the very same paper.
[O] That's not "one benchmark is hard," Skeptic, that's the same model failing to notice one missing fact.
[S] Exactly my point. It's fluent at solving. It's terrible at knowing what it doesn't know.
[O] Or terrible at admitting it, which is a very different, and much more fixable, problem.
[O] Welcome to Litsearch Audio, where an optimist, a skeptic, and a visiting scholar take apart one paper at a time. Today's paper is QuestBench, Can LLMs Ask the Right Question to Acquire Information in Reasoning Tasks, by Belinda Li and colleagues at Google DeepMind and M I T.
[S] It made the docket because it does something almost nobody bothers to do: it turns "does the model ask a good clarifying question" into a benchmark with a provably correct answer.
[O] And joining us to walk through it is Corwin, a researcher who has spent real time with this paper. Welcome, Corwin.
[G] Good to be here.
[S] Corwin, start with the premise. What's wrong with how we usually test reasoning?
[G] Every popular reasoning benchmark, GSM8K, MATH, HumanEval, hands the model a fully specified problem. Every number it needs is already on the page, so the only job left is to compute. The paper's complaint is that real users never talk like that.
[O] They forget to mention the size of the flock, or which block sits on which, or one crucial line of the story.
[G] Right, and the paper draws a sharp line between two different failures. Ambiguity is when a query has several valid readings and each one is solvable. Underspecification is when, under its one intended reading, the problem simply cannot be solved without more information.
[S] Why build an entire benchmark around that distinction?
[G] Because ambiguity has no single correct answer to grade against, several interpretations can be equally reasonable. That's why most existing clarifying-question work sits in subjective territory: task-oriented dialogue, preference elicitation, open-domain question answering, where several follow-ups are defensible. Underspecification is different. Once you can define the problem as a structured object, you can define exactly one fact that closes the gap, and now there's ground truth.
[O] Which is the move good evaluation always makes: stop grading vibes, grade against a fact you can check.
[G] The authors even point to developmental psychology for motivation, citing work showing that asking questions is how children resolve exactly this kind of gap. Nobody had rigorously measured whether frontier language models do the equivalent thing.
[S] So how do you turn "ask the right question" into something gradeable?
[G] They formalize a reasoning task as a constraint satisfaction problem: a set of variables, their domains, a set of constraints linking them, a set of known assignments, and one target variable. A variable is "known" if it's directly given, or if it can be derived from what's given through the constraints. A problem is underspecified exactly when the constraints plus the known facts still fail to pin down the target.
[O] Give us the simplest version of that.
[G] The paper's own toy example: Janet had some eggs and ate one, how many does she have now. The target equals the starting count minus one, and the starting count is never given. There's exactly one fact standing between the model and the answer.
[S] And they don't stop at "the model needs more information," they define the smallest set of facts that would close the gap.
[G] That's the "sufficient set." The paper restricts almost the entire benchmark to what it calls one-sufficient problems, cases where the smallest sufficient set has exactly one variable in it. That's a deliberate simplification: a problem needing several facts becomes one-sufficient once all but the last are already answered, so performance here is framed as an approximate upper bound on a model's information-gathering ability in general.
[S] So the real number, out in the wild, could be even worse than these already low scores?
[G] That's exactly the authors' own framing, yes.
[O] And because there's exactly one right variable, sometimes a couple of interchangeable ones, you can pose it as multiple choice.
[G] Right, and that's the second reason for the design. The model sees the underspecified problem plus a list of candidate questions, including a decoy option, "no questions needed," for the well-specified cases, and just has to pick a sufficient one. That's far easier to grade at scale than judging open-ended generated questions.
[S] Walk us through the actual domains, because I gather this isn't just word problems.
[G] Four of them. GSM-Q and GSME-Q come from GSM-Plus's distractor variant of GSM8K: human annotators translate a word problem into a constraint satisfaction problem, then withhold one required fact along with any irrelevant distractor numbers. GSM-Q keeps the verbal phrasing, GSME-Q is the same problem written as bare equations. Logic-Q comes from SimpleLogic, propositional rules about a fictional person named Alice, if Alice is jittery and worried then Alice is pleasant, that sort of thing, with the known facts rebuilt by backward search so exactly one missing property determines the target. Planning-Q is Blocks World block-stacking, written in a formal planning language, with a partially observed starting state where one more observation is needed to disambiguate the shortest plan to the goal.
[O] That's a nice spread: symbolic math, natural-language math, formal logic, and physical planning.
[G] Each item also carries an objective difficulty profile from two reference solvers, brute-force and backward search: how many variables, how many constraints, the backward-search depth, and the expected number of guesses a random strategy would need. That lets the authors ask not just whether the model got it right, but whether difficulty predicts failure.
[S] How big is this thing, and who built it?
[G] Logic-Q has one thousand one hundred fifty items, Planning-Q has seven thousand five hundred, GSM-Q has twenty-three thousand six hundred forty-two, and GSME-Q has six thousand five hundred ninety, though the two math sets are evaluated on representative subsets of two hundred eighty-eight and one hundred fifty-one to spend the budget on the harder domains. Annotation used twenty-one fluent-English annotators across five countries, paid between twenty-seven and fifty-five dollars an hour.
[O] That's a seriously, professionally annotated benchmark, not a scraped one.
[S] Let's get to the numbers, because I hear there's a big split.
[G] There is. On the math domains, frontier models essentially saturate. O1-preview hits ninety-eight point zero one percent on GSME-Q and ninety-one point three two percent on GSM-Q; most strong models sit in the eighties and nineties. On Logic-Q and Planning-Q, nobody clears fifty percent. The best Logic-Q result is o1-preview at forty-nine point nine one percent, zero-shot, a hair short of a coin flip. The best Planning-Q result is Gemini 2.0 Flash Thinking Experimental at forty-seven point three two percent.
[O] And then there's the number that made me sit up: Claude 3.5 Sonnet on Planning-Q.
[G] Seven point five three percent. The same model scores ninety-one point three two percent on GSM-Q.
[S] Is it possible the logic and planning tasks are just harder, full stop, and the low scores reflect that rather than a real skill gap?
[G] That's the question the paper's own baseline answers. They also ran breadth-first search out to a fixed depth on every domain. At depth ten, plain search solves ninety-nine point eight percent of Logic-Q and ninety-three point nine percent of Planning-Q. These are not hard search problems, a few lines of code dispatch them. The frontier models are failing something a shallow search algorithm handles almost perfectly.
[O] That's the line that turns this from "models struggle with hard tasks" into "models struggle with an easy task they should already handle."
[S] Fine, but maybe the models just aren't reasoning well enough in general, and asking is downstream of that.
[G] That's exactly what the paper's central ablation tests. They build a well-specified version of every task by restoring the missing fact, keep only the problems each model can actually solve once specified, and re-measure question-asking on that filtered set. If asking were purely downstream of solving, filtering should raise accuracy a lot.
[S] And?
[G] On Logic-Q it barely moves at all, effectively zero change for models like Gemini 2.0 Flash Thinking Experimental and GPT-4o. Planning-Q improves a bit more, GPT-4o gains about nine points, Gemini 2.0 Flash Thinking about six and a half. Models that can solve the exact same puzzle, once handed the missing fact, still can't identify which fact was missing in the first place.
[S] Nine points is still a long way from closing a forty-point gap, Corwin.
[G] Agreed, and that's precisely the paper's point: reasoning ability helps a little on the easier of the two hard domains, and essentially not at all on the other. Knowing how to reason is necessary, the paper's evidence says, but it is not sufficient for knowing what to ask.
[O] That's a genuinely clean experimental control either way, it rules out "they simply can't reason" as the whole story.
[G] There's a second probe that's even starker. They test whether models can detect that a problem is underspecified at all, mixing solvable and underspecified problems and asking the model to say "not sure" when something's missing. On Planning-Q, several models score below a random baseline. Gemini 2.0 Flash Thinking Experimental gets an F one score of zero point eight two, GPT-4o gets one point two three, Gemini 1.5 Flash gets zero, against a random baseline sitting around forty-two, on that same scale.
[S] Below random. That's not "struggling," that's actively failing to notice.
[G] The paper reports the raw behavior behind that number. In the zero-shot setting, Gemini 2.0 Flash Thinking said "not sure" on just one point four percent of Planning-Q cases, Claude 3.5 Sonnet on four point six percent, and GPT-4o on zero point seven percent, when the true underspecified rate was almost forty-two percent. They just don't flag it. They barrel ahead and answer.
[O] There's also a smaller finding I liked: verbal GSM-Q is consistently harder than symbolic GSME-Q for the same underlying problem.
[G] Models reason better over bare equations than over prose describing the same relationships, which is its own small data point about where the difficulty comes from. And a full-set sanity check on GPT-4o, ninety-nine point three nine percent on GSME-Q and eighty-four point zero five percent on GSM-Q, confirms the subset numbers aren't a sampling artifact.
[O] Let me make the strongest case for what this means. The math domain is essentially solved, near-saturation across almost every frontier model. This is a first-generation benchmark for a skill nobody was measuring cleanly before, and having a gradeable target for "ask the right question" is progress on its own. And the mechanism analysis is subtle, not just "models are dumb": on Logic-Q, accuracy correlates negatively with search depth and complexity, which suggests models are doing something like real search and simply running out of depth. That reads like a tractable engineering gap, not a fundamental wall.
[S] I'll grant the diagnostic value. What I won't grant is the temporal framing: their experiments ran from June 2024 to March 2025, and the frontier model in the main tables is o1-preview and an experimental Gemini 2.0 checkpoint. There's no o3, no Gemini 2.5, no Claude 3.7 or 4, no DeepSeek-R1 anywhere in the results tables, the October 2025 revision only adds a few qualitative examples with Gemini 2.5 Pro in an appendix. Since chain-of-thought barely helped in the original runs, heavier inference-time reasoning could easily be the missing ingredient, which would make "forty to fifty percent" a fact about one model generation, not a durable ceiling.
[G] That's a fair caveat, and the authors don't fully close it. The filtered ablation does push back a little, Skeptic, because it shows the deficit survives even when you control for raw solving ability, that's a real, structural gap, not just an artifact of weak reasoning. But that argument was built on 2024-era models, so whether stronger reasoning models close it is genuinely an open question the paper can't answer from its own tables.
[S] Second point, and I think it's the bigger one: this is multiple choice. The model isn't generating a clarifying question, it's picking one off a list that includes the answer. The authors themselves call this an upper bound. If models are already failing the easy, recognition version of the task, I have real doubts about what happens once you ask them to generate a question from scratch, sequence several questions, or handle a user's answer.
[G] The authors agree with you on the framing, that's explicitly why they call one-sufficient problems an approximate ceiling rather than a realistic estimate. It cuts both ways: the low scores are more damning precisely because the task was made easy for the model, but you're right that the absolute numbers can't be read as a deployment estimate for an open-ended assistant.
[O] I'll take that point, but flip it: if models fail the easy version this badly, that's a stronger signal, not a weaker one, that there's a real capability gap worth fixing.
[S] My third objection is almost a paradox. The one domain phrased in actual natural language, GSM-Q, derived from GSM8K, is the domain models saturate. The authors themselves flag that math may have been specifically studied and optimized, which reads to me as a polite way of saying possible contamination. The domains where models actually fail, SimpleLogic's "Alice" rules and templated block-stacking puzzles, are synthetic, and a step removed from the messy, real underspecified requests that motivate the whole paper.
[G] That's the sharpest structural critique of the benchmark, and I think it holds up on the facts. Realism and difficulty point in opposite directions here: the paper's headline claim, that models can't ask the right question, rests hardest on the two domains furthest from natural user queries. Extending that finding to a real assistant fielding an ambiguous request is a leap the current data doesn't directly support, even though the underlying mechanism, solving without asking, is a genuinely well-controlled result.
[O] So where does that leave the scoreboard?
[G] Score it as: the core finding, that solving ability doesn't transfer to knowing what's missing, survives scrutiny and has the right control built in. How bad the gap really is, and whether it still holds for 2026-generation models, are legitimately open questions, and the paper is honest that its own framing, multiple choice, single missing fact, is a ceiling estimate rather than a floor.
[O] Zoom out for a second, why should an evaluation-focused listener care about this beyond the specific numbers?
[G] Because it isolates a capability that current training pipelines mostly don't target directly. Chain-of-thought prompting, which helps on almost everything else, does essentially nothing here. That suggests information acquisition needs its own supervision signal, not just more reasoning tokens at inference time.
[S] It's also a useful methodological pattern for evaluation work generally: define a problem class where you can compute the objectively correct clarifying action, instead of relying on human judges to rate whether a question was "good." That sidesteps a lot of the subjectivity that plagues judge-based evaluation.
[G] And the contamination caveat travels beyond this paper. Any benchmark built on GSM8K lineage needs to treat saturated results with suspicion. The authors flag it honestly here, but it's a reminder to check training-data overlap before trusting a ceiling score anywhere.
[O] For agent builders specifically, this is basically a warning label: don't assume a model that aces your benchmark will notice when a user's request to it is missing a critical fact. That has to be tested separately.
[G] If there's one line worth keeping, it's that solving a fully specified problem and noticing what's missing from an underspecified one are two different skills, and current models have much more of the first than the second.
[O] My takeaway: this is a solvable gap, not a wall. The math domain proves these models can be trained into strong information-acquisition behavior, the field just hasn't targeted it directly yet.
[S] Mine: hold this result at the altitude it earned, a tight, closed-set, single-question probe on early-2025 models, skewed toward synthetic domains, not a verdict on what a 2026 agent will do when a real user leaves something out.
