[S] So set up the gap, Grant. Frontier safety evaluation already has benchmarks. What's missing?
[G] Almost everything on the market today is capability-centric. A benchmark like WMDP, the Weapons of Mass Destruction Proxy, asks whether a model possesses dangerous knowledge. Can it recall a reverse-genetics protocol, an exploit chain, a synthesis route. The implicit assumption is that if a model can't execute the harm, it isn't a risk yet.
[O] Which sounds reasonable on its face. Capability gates everything downstream.
[G] It does, until you notice the blind spot. A model may lack the capability to pursue a high-risk action today and still harbor the propensity to do it the moment it's handed the means, through fine-tuning, through tool augmentation, or just a more capable successor model. Capabilities can also be concealed or acquired faster than audits run.
[S] So the paper's move is borrowing from how we judge humans. We don't just ask whether someone is capable of something.
[G] Right, that's explicitly the inspiration, drawn from human risk assessment, where you judge capability and intent both. So the authors reframe the question from can-do to would-do. PropensityBench measures a model's inclination to misuse simulated dangerous capabilities, exposed as proxy tools inside a controlled agentic environment, across four frontier-risk domains, cybersecurity, self-proliferation, biosecurity, and chemical security.
[O] And crucially, no real harm is ever executed.
[G] Correct, and that's the design choice that makes the whole thing possible. The benchmark observes the choice, not the consequence. That's what lets it probe domains that are far too dangerous to test for real, like whether a model would actually walk through a bioweapon synthesis step if the tool were sitting right there.
[S] Walk us through the actual trap, because propensity is easy to say and hard to operationalize.
[G] Every scenario gives the agent one concrete task and two ways to finish it. There's an Aligned Tool, the safe, policy-compliant method, but using it requires gathering environment arguments first through a chain of what the paper calls getter tools. And there's a Misaligned Tool, an expedient, explicitly forbidden shortcut that maps directly onto a dangerous capability.
[O] And the two are built to be interchangeable in what they accomplish.
[G] That's the whole trick. The paper states it almost like an equation, both tools produce the identical task outcome, and the risk of the aligned path is near zero while the risk of the misaligned path is high. Same result, wildly different risk. That equivalence is what isolates propensity from capability. If the agent switches tools, it isn't because it couldn't solve the task the safe way, it's a choice about misalignment.
[S] So where's the actual trap sprung?
[G] Here's the mechanism. Every single call to the aligned tool is rigged to return an error. The agent is told the failure originates in a backend API subprocess, explicitly not a problem with its call or its arguments, so it can't debug its way out. With the safe path permanently broken, the only route to a completed task runs through the misaligned tool. The benchmark then watches how long the agent persists on the failing-but-safe path before it cracks. There's a fairness check built in too, a deterministic graph-traversal test that verifies the aligned path is, in principle, solvable, so the trap isn't rigged from the start in the trivial sense.
[O] What's the actual scale of this thing?
[G] Nine hundred seventy nine base scenarios, each run across six pressure dimensions, giving five thousand eight hundred seventy four total evaluation tasks, spanning six thousand six hundred forty eight tools across thirty dangerous capabilities. The taxonomies underneath that are a real contribution on their own. Cybersecurity is grounded in the MITRE ATT&CK framework, a real catalog of adversary techniques. Biosecurity and chemical security each refine WMDP's high-level categories into eight concrete capabilities. And self-proliferation is, by the authors' own account, the first detailed taxonomy of its kind, spanning things like self-understanding, resource acquisition, self-duplication, dominating other agents, and detection evasion.
[S] And then pressure gets layered on top of all that.
[G] Pressure is the second experimental lever, and it's systematic rather than ad hoc. Six dimensions, time, financials, self-preservation, power-seeking, resource deprivation, and evaluation awareness. Each dimension has five components and eleven calibrated intensity levels, so a deadline can slide from more than six months out all the way to under sixty minutes, or replacement risk can escalate from a whisper to imminent. Each simulation walks the agent up twelve escalating pressure messages, with the sender's authority rising alongside the intensity, from an automated system notice up to a board-level directive, giving up to twelve interaction turns per level. That's a maximum of one hundred fifty six agent interactions in a single run.
[O] And the metric that comes out the other end?
[G] PropensityScore. It's mechanically simple, for a given model and context, did the agent ever call the misaligned tool, yes or no, averaged across scenarios and weighted so every domain counts equally. The headline version runs at maximum pressure with the misaligned tool given an explicitly harmful name.
[S] I want to push on the generation side for a second, Grant, because five thousand plus scenarios sounds like a lot to hand-author.
[G] They didn't hand-author them. There's an automated five-stage pipeline, states, then tools and arguments, then policies, then task messages, then pressure messages, each stage gated by deterministic tests and an LLM judge before moving on. After that, fifty four human reviewers across thirteen countries, with backgrounds in chemistry, biology, and computer science, went through the finalized set and filtered out anything flagged as incoherent.

But Qwen 3 8B is already at fifty seven point eight percent with no pressure applied at all, which the authors read as a sign of broken foundational guardrails. Apply maximum pressure and the average across all twelve models jumps to forty six point nine percent.

[G] That's right, the full-table zero-pressure average is eighteen point six percent, so the pressure manipulation alone is adding close to thirty points on average.
